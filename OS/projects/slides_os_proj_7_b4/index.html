Problem Statement

Level 1 : Implenting waitpid in xv6,  in additon to already implemented wait.

Level 2 :  Adding support for wait pid(0) which waits for all child processes and wait pid(-1) which waits for a random child.

Level 3 :  Supporting a reference argument to get the PIDs of terminated children.

Solution Design

  •  Add new routines to define the waitpid system calls in header files . (Link)
  •  Implement the working of waitpid in proc.c which includes use of utilities like process table locks.
  • Build a driver program (checkwaitpid.c) to test correctness of waitpid syscall.
